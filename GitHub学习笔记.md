 ### 一、我理解的 Geek 与 Geek 精神
- **Geek**：原指“极客”，是对技术、创新有极致热爱，喜欢钻研、折腾的一群人。他们不只是用工具，更想弄懂原理、创造新东西。
- **Geek 精神**：
  - 好奇：对未知充满探索欲，爱问“为什么”
  - 分享：乐于把自己的发现、代码、经验分享给社区
  - 折腾：不怕踩坑，喜欢动手实践，从错误中学习
  - 自由：崇尚开源、开放，追求技术的自由与透明
  ### 二、GitHub 与核心概念理解
- **GitHub 是什么**：
  它是一个基于 Git 的代码托管平台，也是全球最大的开源社区。我们可以在这里存放代码、协作开发、学习别人的项目。
- **Fork**：
  把别人的仓库“复制一份”到自己的 GitHub 账号下，相当于“先拿过来自己改”，不会影响原项目。
- **Pull Request（PR）**：
  当你在自己 Fork 的仓库里改完代码后，可以向原项目发起“请求”，让原作者把你的修改合并到主项目里，是开源协作的核心流程。
  *学习来源*：豆包，谷歌百度，B站

  ### 四、文档整洁规范的说明
我遵循了这些 Markdown 规范，让文档更清晰：
1. **标题分级**：用 `#` `##` `###` 区分大标题/小标题，结构清晰
2. **列表使用**：用 `-` 或 `1.` 罗列要点，避免大段文字
3. **代码块**：用 ``` 包裹代码/命令，方便阅读
4. **强调**：用 **加粗** 突出关键词
5. **段落分隔**：空行分隔不同模块，避免拥挤
