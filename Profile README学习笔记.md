# GitHub Profile README 学习笔记

## 一、参考案例

### 1. 案例 1：abhchand
- **链接**：https://github.com/abhchand
- **风格**：极简、干净、专业、信息密度高。
- **参考亮点**：
  - 一句话清晰自我介绍。
  - 整齐排列的技能徽章。
  - 简洁但有层次的链接布局。
- **学习重点**：少即是多，突出核心信息。

### 2. 案例 2：kyechan
- **链接**：https://github.com/kyechan
- **风格**：生动、动态、好玩、信息丰富。
- **参考亮点**：
  - 自动生成 GitHub 数据统计卡片。
  - 动态生活模块（如正在听的歌）。
  - 大量 Emoji 增强可读性但不失逻辑。
- **学习重点**：善用插件，让主页更“活”。

## 二、学到的技巧

### 1. 仓库命名规则（必须遵守）
- 仓库名必须 **完全与 GitHub 用户名一致**，不然会看不到。
- 必须设为 **Public（公开）**。
- 必须勾选 **Add a README file**。

### 2. Markdown 排版技巧
- 标题层级：`#` 越大
- 列表：`-` 与 `1.`
- 强调：`**粗体**` `*斜体*`
- 代码块：``` 代码 ```
- Emoji：用于分区、增强视觉效果

### 3. 美化工具
- **技能徽章（shields.io）**
  - 可生成各种好看的技术栈图标。
  - 自定义颜色、Logo、风格。
- **数据统计（github-readme-stats）**
  - 自动生成贡献图、语言占比。
  - 多主题可选。
