# 🔧 GitHub 仓库管理与 Git 协作问题

## 📁 一、GitHub 文件夹创建

**📌 重要**

GitHub 不支持创建空文件夹。每个文件夹必须至少包含一个文件，常用做法是在空文件夹中放一个空的 .gitkeep 文件作为占位。

🖥️ 网页端创建文件夹步骤

- 进入 GitHub 仓库，点击 Add file → Create new file
   
-  在文件名栏输入：
   ```
   文件夹名/.gitkeep
   ```
   例如：第一阶段/.gitkeep
- 下拉页面，点击 Commit new file
      → 仓库会自动生成对应的文件夹（内含 .gitkeep）

**💡 小技巧**

- 也可以先创建文件夹再上传文件，但上述方法最直接。
- 如果想创建多层嵌套文件夹，直接输入 父文件夹/子文件夹/.gitkeep 即可一步到位。

## ⚠️ 二、Git Bash 常见易错点

1️⃣ 粘贴命令不能用 Ctrl + V

在 Git Bash 中，Ctrl + V 并不生效，粘贴的正确方式是：

· 鼠标右键 → 选择 Paste
· 或按 Shift + Insert 快捷键

💡 同样，复制也可以用右键 → Copy，或 Ctrl + Insert。

2️⃣ Git 命令中空格是关键

每个 Git 命令的各个部分之间必须用空格分隔，否则会被视为一个整体导致报错。

```bash
# ✅ 正确写法（注意空格）
git config --global user.name "池海秀"
```

```bash
# ❌ 错误写法（缺少空格）
git config--global user.name"池海秀"
```

🧠 理解：git、config、--global、user.name、"池海秀" 是五个独立的部分，空格是它们的分隔符。

## 🧩 三、Git 协作常见问题与解决

### 1. SSH 连接 GitHub 被拒绝

**问题现象**
执行 ssh -T git@github.com 时提示：

```
ssh: connect to host github.com port 22: Connection refused
```

>原因:
>所在网络屏蔽了 SSH 默认的 22 端口，导致无法使用 SSH 协议连接 GitHub。

>解决方法
>改用 HTTPS 协议进行 Git 操作，并使用 Personal Access Token 代替密码。

- 克隆仓库时使用 HTTPS 地址：
   ```bash
   git clone https://github.com/团队名/仓库名.git
   ```
-  推送时，用户名填 GitHub 用户名，密码填生成的 Personal Access Token（而非登录密码）。

---

### 2. 无法直接推送代码到团队仓库

**解决方法**
>使用 Fork + Pull Request 工作流：

- Fork 原仓库到自己的 GitHub 账号下。
- 克隆自己的 Fork 仓库到本地。
-  创建分支，在分支上修改文件。
-  推送分支到自己的 Fork 仓库。
- 在 GitHub 上从自己的 Fork 分支向原仓库的 main 分支发起 Pull Request。

---

💡 详细步骤可参考：
```
git clone https://github.com/你的用户名/仓库名.git
cd 仓库名
git checkout -b feat/你的名字
# 修改文件
git add .
git commit -m "你的提交信息"
git push -u origin feat/你的名字
然后在原仓库页面创建 PR。
```
---

📌 小结

· 创建文件夹时，牢记 .gitkeep 占位法。

· Git Bash 的快捷键与 Windows 常规不同，多使用右键或 Shift+Insert。

· 命令中的空格是语法的一部分，写错就会报错。

· SSH 被拒时，改用 HTTPS + Token。

· 无写权限时，用 Fork + PR 参与协作。
