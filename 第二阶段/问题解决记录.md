# GitHub README索引制作+文件夹创建
 
## 一、标准项目文件夹创建方法
 
 > GitHub不支持创建空文件夹，所有文件夹必须包含至少1个文件（可新建空的 **.gitkeep**文件占位）
 
网页端创建文件夹步骤
 
1. 进入GitHub仓库，点击Add file → Create new file
2. 文件名栏输入： 文件夹名/.gitkeep （例： src/.gitkeep ）
3. 下拉点击Commit new file，自动生成对应文件夹

### （二）索引跳转核心原理
1. Markdown标题会自动生成**锚点**，格式为`#标题内容`
2. 索引语法：`[链接文字](#锚点名称)`
3. 特殊情况：标题含空格/符号，锚点自动转为**短横线连接**，例：`## 快速开始` → 锚点`#快速开始`

### （三）索引优化技巧
1. 层级索引：二级目录下可加三级索引，例：`- [安装步骤](#安装步骤)`
2. 返回顶部：在每个章节末尾添加`[⬆ 返回顶部](#目录)`，方便快速跳转
```markdown
[⬆ 返回顶部](#目录)
 
 
三、提交文件夹+README到GitHub
 
1. 本地创建好文件夹和README.md后，执行Git命令
 
bash
  
git add .
git commit -m "feat: 初始化项目结构+添加README文档"
git push origin main
 
 
2. 推送完成后，GitHub仓库会显示完整文件夹和规范README文档
