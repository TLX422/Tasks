## 问题一：Markdown 链接添加
 **问题描述**：不了解如何在 README 中添加可点击的链接，实现文件跳转。  
 
解决方案：
- 核心语法： [显示文字](链接地址) 
- 本地文件链接：使用相对路径  ./文件名.md 
- 外部网页链接：直接使用完整 URL
- 避坑要点：路径大小写、空格需与文件名完全一致
- 示例代码：
[GitHub学习笔记](./GitHub学习笔记.md)
[在线笔记](https://github.com/用户名/仓库名/blob/分支名/GitHub学习笔记.md)
 
## 问题二：404 页面未找到错误
**问题描述**：点击 README 中的链接时出现  404 - 页面未找到 ，提示“该分支不包含路径 GitHub学习笔记.md”。  

解决方案：  

1. 检查路径与文件名：确认链接路径与仓库实际文件一致（大小写、空格、层级）
2. 确认分支：确保文件存在于当前分支（如  主要角色  分支）
3. 提交推送文件：本地文件需执行  git add/commit/push  推送到远程仓库
4. 复制真实路径：在文件页面通过「Copy path」或地址栏获取正确路径

## 问题 三：外部文件引入 README
 
**问题描述**：希望将外部文件（本地或在线）链接到 README 中。
解决方案：
 
- 本地文件：先上传到 GitHub 仓库，再使用相对路径链接
- 在线文件：直接使用完整 URL 作为链接地址

## 问题 四：复制文件真实路径
 
**问题描述**：不知道如何获取文件的真实路径，避免链接 404 错误。
解决方案：
 
1. 方法一（浏览器地址栏）：进入文件详情页，复制地址栏 URL 作为完整链接
2. 方法二（复制相对路径）：在文件页面点击「...」→「Copy path」，获取相对路径后拼接  ./
